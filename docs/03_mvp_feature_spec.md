# 보드게임 기록 서비스 — 3단계 요구사항 정의 및 기능 명세

> 목적: 기능 팽창을 막는 단계. 1단계(문제: 기록 파편화), 2단계(포지셔닝: 통합 + 마찰 제로)에서 살아남은 것만 MVP에 넣는다.
> **근거의 한계**: 페르소나 B 인터뷰(5~10명)는 미실시. 아래 3개 답변은 개발자 본인(하드게이머, n=1)의 자가 응답이며, 정식 인터뷰의 대체가 아니라 잠정 근거다. MVP 출시 전 또는 직후 최소 5명 검증을 유지 과제로 남긴다.
>
> | 질문 | 답변 (n=1) | 설계 반영 |
> |---|---|---|
> | BG Stats 한글화 이후에도 왜 안 쓰는가 | 기록하기 자체가 귀찮음 | 마찰 제로가 최우선 설계 원칙 |
> | 기록·컬렉션이 한 앱이면 옮길 것인가 | 기존 데이터를 옮길 수 있다면 옮긴다 | **데이터 임포트를 Must로 승격** |
> | 기록에 10초 이상 걸리면 포기하는가 | 입력 항목이 많으면 장벽이 높다 | 필수 입력 2개(게임, 날짜=자동)로 제한, 나머지 전부 선택 |

---

## 1. 설계 원칙 (기능 판정 기준)

모든 기능 제안은 아래 3개 질문을 통과해야 MVP에 들어온다. 하나라도 "아니오"면 Should 이하로 강등.

1. **기록 루프에 기여하는가?** — "플레이 종료 → 10초 기록 → 컬렉션/통계 자동 갱신" 루프를 빠르게/자주 돌게 만드는가.
2. **전환 장벽을 낮추는가?** — BG Stats/BGG/메모앱에서 넘어오는 비용을 줄이는가.
3. **혼자서도 가치가 있는가?** — 다른 유저가 0명이어도 작동하는가. (커뮤니티·매칭류 자동 탈락 — 콜드스타트 회피)

## 2. MoSCoW 분류

### Must (이것 없으면 출시 안 함)

| # | 기능 | 근거 | 범위 제한 (팽창 방지) |
|---|---|---|---|
| M1 | **10초 플레이 기록** | 마찰이 1번 적 (자가응답 1·3) | 필수 입력은 게임 선택뿐(날짜 자동). 플레이어·점수·승자·메모·사진 전부 선택 입력. "저장" 없이 뒤로가기 시 임시저장 |
| M2 | **한글 게임 검색** | 한글 네이티브 축 | 한글명·영문명·초성 검색. 별칭/줄임말 DB는 Could로 미룸. 최근 기록 게임 상단 고정 |
| M3 | **컬렉션 관리 (보유/위시)** | 통합 축의 절반 | 보유·위시 2개 상태만. 확장판 연결·구매가·에디션 구분은 Could |
| M4 | **기록↔컬렉션 자동 연동** | 통합 축의 핵심, BG Stats가 못 하는 것 | 기록 시 미보유 게임이면 "컬렉션에 추가?" 1탭 제안. 컬렉션 화면에 게임별 플레이 횟수·최근 플레이·**노플 배지** 자동 표시 |
| M5 | **기본 통계** | 기록의 보상. 이게 없으면 기록할 이유가 없음 | 총 플레이 수, 월별 추이, 최다 플레이 게임 top 5, 노플 게임 수. H-index·히트맵·플레이어 비교는 Should 이하 |
| M6 | **데이터 임포트 (BG Stats / BGG)** | 전환의 전제조건 (자가응답 2). 이것이 없으면 타겟 유저가 옮기지 않음 | BG Stats 내보내기 파일(JSON/CSV) 업로드 + BGG 컬렉션·플레이 단방향 가져오기. **양방향 동기화는 Won't** — 파편화를 다시 만들기 때문 |
| M7 | **계정·백업** | 데이터 유실 = 이 서비스에선 치명상 (컬렉션 앱 리뷰의 기기 이전 불만) | 소셜 로그인 1종 + 서버 저장. 멀티디바이스 실시간 동기화는 Should |
| M8 | **게임 DB (BGG 연동)** | 검색·표지 이미지 기반 | BGG API 온디맨드 조회 + 자체 캐시 + 한글명 매핑 테이블. 전체 DB 사전 복제하지 않음 |

### Should (MVP 직후 1~2 스프린트, 리텐션 훅)

| # | 기능 | 근거 |
|---|---|---|
| S1 | 사진 첨부 기록 | 인스타 대체재. 마찰 낮은 감성 기록 |
| S2 | **기록 카드 공유 이미지** (레포브식) | 커뮤니티를 만들지 않고 커뮤니티에 얹혀가는 유일한 성장 채널 |
| S3 | 연말/월말 결산 화면 | "올해 뭘 얼마나 했지"가 원 문제 정의의 출발점. 단, 데이터가 쌓여야 의미 있어 출시 시점엔 불필요 |
| S4 | 컬렉션 필터 (인원수·플레이시간) | BGG 앱 리뷰 최다 요청. 추천 엔진이 아닌 정렬·필터로 한정 |
| S5 | 멀티디바이스 동기화 | M7의 확장 |

**S2 서버 스코프 판정 (2026-07-20, 갱신):** 단건 플레이 1회 카드, 데이터는 게임명·날짜·썸네일로 한정 — 카드 렌더링·공유 자체는 업계 표준(Strava, Letterboxd 계열)과 동일하게 Flutter 클라이언트 단독 처리. 다만 "기록 직후"뿐 아니라 **히스토리에서도 나중에 공유 가능**해야 한다는 요구가 확인되면서, M1에서 의도적으로 미뤘던 `GET /api/v1/plays` 목록 조회(`04_system_design.md`에 이미 설계돼 있던 스코프)를 구현 — 상세는 `play/CLAUDE.md`. 공유 가능 기간 제한은 두지 않음(성능·보안 어느 쪽으로도 근거 없음, 순수 UX 판단이라 서버 설계에 반영 안 함). 공유 이벤트 서버 추적은 여전히 보류(관측된 지표 수요 없음, ADR-002 정신).

### Could (여유 있으면, 아니면 안 함)

- 심층 통계 (H-index, 히트맵, 플레이어별 전적 비교)
- 게임 별칭·줄임말 검색, 태그, 플레이 위치 기록
- 확장판·에디션 구분, 구매가 기록
- 컬렉션 필터 확장: 난이도(weight)·메커니즘. BGG 데이터(M8, 현재 보류) 없이는 유저 수동 입력이 필요해 마찰이 큼 — M8 해제 후 재검토
- 다크모드, 위젯

### Won't (이번 릴리스에서 명시적으로 안 함 — 2단계 포지셔닝 경계 재확인)

| 기능 | 이유 |
|---|---|
| 룰 요약·AI Q&A | 저작권 리스크, 보드게임마스터 영역 |
| 모임 매칭 | 양면 콜드스타트, 보드후드 영역, 원칙 3 위배 |
| 커뮤니티 (피드·댓글) | 보드라이프와 경쟁 불가, 원칙 3 위배. S2(공유 이미지)로 대체 |
| 취향 기반 추천 엔진 | 1단계에서 탈락한 '게임 셀렉터'의 재림. S4 필터로 충분 |
| BGG 양방향 동기화 | 파편화 재생산 + API 불안정 리스크(BG Stats도 "BGG 변경으로 기능 중단 가능" 고지 중) |
| 게임 리뷰·평점 시스템 | 개인 기록용 별점(선택 입력)으로 충분. 공개 평점은 커뮤니티 기능 |
| 가격 정책 확정 | 2단계 결론 유지 — 인터뷰 전까지 완전 무료로 출시, 과금은 열린 질문 |

## 3. 유저 스토리 & 인수 조건

### Epic 1: 기록 (M1, M2)

**US-1.1** As a 기록 덕후 유저, I want 게임이 끝난 자리에서 10초 안에 플레이를 기록하고 싶다, so that 귀찮아서 기록을 포기하는 일이 없다.

인수 조건:
- [ ] 홈 화면에서 기록 시작까지 1탭
- [ ] 게임 선택 후 즉시 저장 가능 (다른 입력 없이) — 이때 날짜는 오늘, 나머지 필드는 비움
- [ ] 최근 기록한 게임 5개가 검색 없이 상단 노출, 탭 1회로 선택
- [ ] 게임 선택 → 저장까지 신규 유저 기준 10초, 재방문 유저(최근 게임 탭) 기준 5초 이내 (실기기 측정)
- [ ] 저장 완료 시 해당 게임의 누적 플레이 횟수가 토스트/스낵바로 표시 ("카탄 12번째 플레이!")

**US-1.2** As a 유저, I want 점수·플레이어를 넣고 싶을 때만 넣고 싶다, so that 입력 항목이 많아 기록을 미루지 않는다.

인수 조건:
- [ ] 플레이어·점수·승자·별점·메모·사진은 모두 선택 입력이며, 비워도 저장에 아무 경고가 없다
- [ ] 플레이어 추가 시 최근 함께한 플레이어가 자동 제안된다 (탭 1회 추가)
- [ ] 점수 입력 시 승자가 자동 판정된다 (수동 변경 가능)
- [ ] 기록 도중 앱 이탈 시 작성 중이던 내용이 임시 저장된다

**US-1.3** As a 한국 유저, I want 한글로 게임을 검색하고 싶다, so that 영문명을 몰라도 기록할 수 있다.

인수 조건:
- [ ] "카탄", "Catan", "ㅋㅌ" 모두 같은 게임을 반환
- [ ] 검색 결과에 한글명(있으면)과 표지 이미지가 표시
- [ ] DB에 없는 게임은 "직접 등록" (이름만으로 로컬 게임 생성)이 검색 결과 하단에 항상 제공 — 검색 실패가 기록 포기로 이어지지 않게

### Epic 2: 컬렉션 & 통합 (M3, M4)

**US-2.1** As a 컬렉션 40종 보유 유저, I want 보유 게임과 위시리스트를 관리하고 싶다, so that 내가 뭘 갖고 있는지 한눈에 본다.

인수 조건:
- [ ] 게임을 보유/위시로 추가·제거 가능
- [ ] 컬렉션 화면에서 보유 수·위시 수가 표시
- [ ] 정렬: 이름순 / 최근 플레이순 / 플레이 횟수순

**US-2.2** As a 유저, I want 기록과 컬렉션이 자동으로 연결되길 원한다, so that 두 앱을 오가던 수고가 사라진다.

인수 조건:
- [ ] 미보유 게임 기록 저장 시 "컬렉션에 추가할까요?" 1탭 제안 (무시 가능, 다시 묻지 않음 설정)
- [ ] 컬렉션의 각 게임에 플레이 횟수·최근 플레이 일자가 자동 표시
- [ ] 보유 후 한 번도 기록이 없는 게임에 "노플" 배지가 자동 표시
- [ ] 컬렉션에서 게임 탭 → 그 게임의 기록 목록으로 바로 이동

### Epic 3: 통계 (M5)

**US-3.1** As a 통계 보는 걸 즐기는 유저, I want 내 플레이가 쌓이는 걸 눈으로 보고 싶다, so that 기록할 동기가 유지된다.

인수 조건:
- [ ] 총 플레이 수, 이번 달 플레이 수, 최다 플레이 top 5, 노플 게임 수가 통계 탭에 표시
- [ ] 월별 플레이 추이 차트 (최근 12개월)
- [ ] 기록 0건일 때 빈 차트 대신 첫 기록 유도 화면 표시 (3주 내 이탈 리스크 대응)

### Epic 4: 전환 (M6, M7)

**US-4.1** As a BG Stats에 기록이 쌓여 있는 유저, I want 기존 데이터를 그대로 가져오고 싶다, so that 갈아타는 순간 과거 기록을 잃지 않는다.

인수 조건:
- [ ] BG Stats 내보내기 파일(.bgsplay/JSON) 업로드 시 플레이 기록(게임·날짜·플레이어·점수)이 이관된다
- [ ] BGG 아이디 입력 시 컬렉션(보유/위시)과 플레이 기록을 가져온다 (단방향, 1회성 + 수동 재실행)
- [ ] 임포트 결과 요약이 표시된다 ("플레이 312건, 게임 87종 가져옴, 매칭 실패 3건") — 실패 항목은 수동 매칭 가능
- [ ] 중복 임포트 시 기존 기록과 중복 병합된다 (중복 생성 금지)
- [ ] 온보딩 첫 화면에서 임포트 경로가 노출된다 ("빈손으로 시작" vs "데이터 가져오기")

**US-4.2** As a 유저, I want 내 기록이 서버에 안전하게 있길 원한다, so that 폰을 바꿔도 데이터가 유지된다.

인수 조건:
- [ ] 소셜 로그인 1종으로 가입 30초 이내
- [ ] 기록·컬렉션이 서버에 저장되며, 재설치 후 로그인 시 전부 복원
- [ ] 내 데이터 내보내기(JSON/CSV) 제공 — 들어올 자유만큼 나갈 자유도 보장 (신뢰 = 전환 장벽 완화)

## 4. 핵심 유저 플로우 (화면 플로우)

### 플로우 A — 온보딩 (전환 유저 vs 신규)
```
앱 실행 → 소셜 로그인 → 분기:
  ├─ "데이터 가져오기" → BG Stats 파일 업로드 / BGG ID 입력 → 임포트 요약 → 홈 (통계가 이미 채워진 상태 = aha moment)
  └─ "빈손으로 시작"   → 홈 (첫 기록 유도 화면)
```

### 플로우 B — 10초 기록 (핵심 루프)
```
홈 → [+ 기록] (1탭)
  → 게임 선택 (최근 게임 1탭 / 검색 / 직접 등록)
  → [바로 저장] ──────────────────→ 저장 완료 (누적 횟수 토스트)
  └→ (선택) 플레이어 추가 → (선택) 점수 → (선택) 메모·사진 → 저장
저장 후:
  ├─ 미보유 게임이면 → "컬렉션에 추가?" 1탭
  └─ 컬렉션·통계 자동 갱신
```

### 플로우 C — 컬렉션 루프
```
컬렉션 탭 → 보유 게임 목록 (플레이 횟수·노플 배지)
  ├─ 게임 탭 → 게임 상세 (기록 목록 + 이 게임 통계) → [이 게임 기록하기]
  └─ [+ 게임 추가] → 검색 → 보유/위시 지정
```

## 5. MVP 범위 요약 (엘리베이터 버전)

**들어가는 것**: 10초 기록, 한글 검색, 보유/위시 컬렉션, 기록↔컬렉션 자동 연동(노플 배지), 기본 통계 4종, BG Stats/BGG 임포트, 계정·백업, 데이터 내보내기.
