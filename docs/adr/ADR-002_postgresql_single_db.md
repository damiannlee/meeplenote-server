# ADR-002: PostgreSQL 단일 DB (Redis·별도 문서 DB 기각)

**Status:** Accepted / **Date:** 2026-07-09

### Context
관계형 코어(유저-게임-기록-플레이어) + 임포트 원본(비정형 JSON) + 한글 검색, 세 가지 성질의 데이터가 있다.

### Options
- **A. PostgreSQL 하나로 전부**: 관계형 + jsonb + pg_trgm
- **B. PostgreSQL + Redis(캐시) + MongoDB(임포트 원본)**: 용도별 최적 저장소

### Decision
**A.** 관계형 스키마가 코어, 임포트 원본은 `jsonb` 컬럼, 검색은 `pg_trgm` 확장.

### 근거
- 트래픽상 캐시가 풀어야 할 문제(핫 리드)가 아직 없음. **캐시는 문제가 관측된 후의 도구**이지 기본 구성요소가 아니다. 캐시 도입 = 무효화 버그라는 새 장애 클래스 도입.
- jsonb는 임포트 원본 보존 요구(재처리 가능성)에 정확히 부합. 문서 DB를 따로 두면 백업·트랜잭션 경계가 2벌이 됨.
- 백업/복구 대상이 1개 = "데이터 유실 제로" 요구에 가장 유리한 구조.

### 뒤집는 트리거
- 통계 조회가 동일 파라미터로 반복 폭주 → 그때 Redis가 아니라 **HTTP 캐시 헤더/ETag부터** 시도, 그래도 부족하면 Redis.
