# ADR-003: BGG 연동 — 온디맨드 프록시 + 로컬 캐시 (전체 미러링·실시간 프록시 기각)

**Status:** Accepted / **Date:** 2026-07-09

### Context
게임 마스터 데이터(이름·인원·시간·썸네일)의 원천은 BGG XML API2. 그러나 BGG API는 레이트리밋이 있고 불안정하며, BG Stats조차 "BGG 변경으로 기능이 중단될 수 있다"고 고지한다. 외부 의존이 핵심 루프(기록)를 막으면 안 된다.

### Options
- **A. 전체 미러링**: BGG DB를 주기적으로 통째로 복제
- **B. 실시간 프록시**: 검색마다 BGG 호출
- **C. 온디맨드 캐시**: 검색 시 로컬 우선, 미스 시 BGG 조회 후 로컬 적재. 이후 그 게임은 로컬에서 서빙

### Decision
**C.** 추가 규칙: BGG 응답이 지연 한도(1.5s)를 넘으면 로컬 결과만 반환하고 백그라운드로 적재. 유저가 실제로 찾는 게임만 캐시가 자람 (한국 유저 수요 반영 DB가 자연 형성).

### 기각 이유
- A: 15만+ 게임 중 한국 유저가 만질 게임은 극소수. 미러링 파이프라인 유지비 + 레이트리밋 리스크가 이득을 초과. 이용약관 리스크도 있음.
- B: BGG 다운 = 검색 다운 = 기록 루프 차단. 레이트리밋 소진도 빠름.

### Consequences
- (+) BGG 장애 시에도 기존 캐시 게임으로 기록 가능. (+) 저장 공간 최소.
- (−) 최초 검색 유저는 콜드 미스 지연 경험 → 인기 게임 상위 N개는 시드 데이터로 선적재해 완화.
- 한글명은 BGG에 없는 경우가 많음 → `name_ko`는 별도 매핑(수동 큐레이션 + 유저 제보)으로 채움. 이것이 이 서비스의 실질 자산이 된다.

> **구현 시점(M2/M8, 2026-07-11) 중요 발견**: 실제 배포 환경에서 라이브 검증한 결과, BGG XML API2(`boardgamegeek.com/xmlapi2/*`)가 현재 **Cloudflare 봇 차단 뒤에 있어 서버 간 HTTP 호출이 401 "Unauthorized. See https://boardgamegeek.com/using_the_xml_api"로 거부된다** (안내 문서 페이지 자체도 Cloudflare JS 챌린지로 막혀있음 — User-Agent 헤더 추가로도 해결 안 됨, 실제 curl로 재현 확인). 이 ADR이 전제한 "무인증 온디맨드 조회"가 현재는 성립하지 않는다.
> 애플리케이션은 BGG 실패(4xx/5xx/타임아웃)를 `BggClient`에서 흡수해 빈 결과로 대체하도록 방어 코드를 넣어뒀기 때문에 **핵심 루프(검색 자체)는 500 없이 정상 동작**하지만, 실질적으로 지금은 BGG 캐싱이 전혀 채워지지 않는 상태다. 재검토 필요 — BGG가 API 키/OAuth 등 공식 인증 경로를 다시 열었는지 확인하거나, 대안(수동 게임 등록 위주로 당분간 운영, 다른 게임 메타데이터 소스 검토 등)을 사용자와 논의할 것.
