# ADR-004: 한글 검색 — pg_trgm + 초성 생성 컬럼 (Elasticsearch 기각)

**Status:** Accepted / **Date:** 2026-07-09

### Context
US-1.3: "카탄", "Catan", "ㅋㅌ" 모두 같은 게임 반환. 검색 대상은 games 테이블(초기 수천~수만 행).

### Options
- **A. LIKE '%q%'**: 인덱스 불가, 그러나 수천 행이면 사실 돌아감
- **B. pg_trgm(GIN) + 초성 생성 컬럼**: 이름 저장 시 초성 문자열을 GENERATED 컬럼으로 파생, 유사도 검색
- **C. Elasticsearch/OpenSearch**: 형태소 분석기(nori) 포함 전문 검색

### Decision
**B.** `name_ko`, `name_en`에 gin_trgm_ops 인덱스, `name_chosung`은 애플리케이션 초성 추출 함수로 생성·저장. 쿼리 판별: 입력이 초성 전용이면 chosung 컬럼, 아니면 trgm 유사도 + 정확 접두 우선 정렬.

### 기각 이유
- A: 지금은 돌아가나 "왜 인덱스를 안 걸었나"에 답이 없음. B와 비용 차이도 미미.
- C: 운영 대상 인프라 +1 (JVM 힙 튜닝, 인덱스 동기화 파이프라인, 버전 업). 게임 "이름" 검색에 형태소 분석은 과잉 — 문서 검색이 아니라 짧은 고유명사 매칭 문제.

### 뒤집는 트리거
검색 대상이 이름을 넘어 설명·리뷰 텍스트로 확장되거나, 오타 교정·동의어 사전이 필요해지는 시점.
