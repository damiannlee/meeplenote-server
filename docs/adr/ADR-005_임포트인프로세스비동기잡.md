# ADR-005: 임포트 처리 — 인프로세스 비동기 잡 (동기 처리·외부 큐 기각)

**Status:** Accepted / **Date:** 2026-07-09

### Context
BG Stats 원본은 수천 플레이(수 MB)일 수 있고, 게임 매칭에 BGG 조회가 섞여 처리 시간이 수십 초~수 분. HTTP 요청 안에서 처리 불가.

### Options
- **A. 동기 처리**: 업로드 요청 안에서 전부 처리
- **B. 인프로세스 큐**: 202 반환, @Async/코루틴 + DB의 import_jobs를 상태 저장소로. 재시작 시 pending 잡 재개
- **C. 외부 큐 (SQS/RabbitMQ/Kafka)**: 별도 브로커 + 워커

### Decision
**B.** 잡 상태를 DB에 두므로 프로세스 재시작에도 잡이 유실되지 않음(재개 가능). 유저당 동시 1잡 제한으로 리소스 폭주 방지.

### 기각 이유
- A: 타임아웃·재시도 시 중복 처리 지옥. 수 분짜리 HTTP는 설계 실패.
- C: 브로커는 "생산자와 소비자가 다른 프로세스/스케일"일 때의 도구. 지금은 둘 다 같은 프로세스이고 처리량이 일 수십 건. 운영 인프라 +1의 근거 없음.

### 뒤집는 트리거 (ADR-001과 연동)
임포트가 API 레이턴시에 영향을 주면 워커 프로세스 분리. 그때도 큐는 **DB 폴링(SKIP LOCKED)** 부터 — 브로커는 그다음.
