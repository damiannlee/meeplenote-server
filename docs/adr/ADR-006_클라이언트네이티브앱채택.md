# ADR-006: 클라이언트 — 네이티브 모바일 앱 채택 (PWA 기각)

**Status:** Accepted / **Date:** 2026-07-09 (2026-07-09 PWA안에서 수정)

### Context
기록은 게임 테이블 위 폰에서 일어난다 → 모바일이 1순위 폼팩터. M1(10초 기록)의 사진 첨부·임시저장·반복 습관 형성이 핵심 요구다.

### Options

| 차원 | A. 네이티브/크로스플랫폼 앱 | B. PWA (모바일 웹) |
|---|---|---|
| 카메라·로컬 저장 안정성 | 완전 지원 | iOS 사파리에서 저장소 축출, 백그라운드 카메라 접근 제약 실재 |
| 반복 사용 습관 | 홈 화면 앱 아이콘 — 마찰 제로 원칙에 부합 | 링크 재방문 의존, 습관화에 불리 |
| 최초 유입 마찰 | 스토어 설치 장벽 | 링크 클릭 즉시 사용 |
| 포트폴리오 서사 | 백엔드(Kotlin)와 언어 통일 가능(Compose Multiplatform) 시 "풀스택 완결성" 서사 | 상대적으로 약한 기술 서사, "리소스 부족으로 인한 타협"으로 읽힐 리스크 |
| 1인 개발 속도 | 스토어 심사·플랫폼 이슈로 상대적으로 느림 | 배포 즉시 |
| 푸시 알림 | 완전 지원 | 웹 푸시 가능하나 iOS 제약 |

### Decision
**A. 네이티브/크로스플랫폼 앱 채택.** 임시저장은 로컬 DB에 영속, 사진은 디바이스 캐시 후 업로드. 앱 아이콘 상시 노출로 기록 습관 형성을 최우선. (구체 스택은 ADR-008에서 Flutter로 확정)

### 기각 이유 (기존 PWA 채택 근거의 재평가)
애초 "보드라이프 링크 유입 → 즉시 체험"이라는 최초 전환 마찰을 근거로 PWA를 택했으나, 재검토 결과 다음 두 가지를 과소평가했다:
1. **사진 첨부·임시저장 안정성**이 iOS PWA에서 구조적으로 약하다는 점 — M1(선택 입력이지만 자주 쓰일 사진 첨부)의 핵심 UX를 흔드는 리스크였는데 최초 결정에서 "Consequences의 부수 리스크"로 축소 평가했다.
2. **반복 습관 형성**이 최초 유입보다 이 서비스의 생존에 더 중요한 지표다(3주 이탈 리스크, 페르소나 B). 최초 전환 마찰을 낮추는 이득보다 습관화를 돕는 앱 아이콘의 이득이 크다고 재평가.
3. 포트폴리오 목적상 클라이언트 플랫폼 선택 자체가 이야깃거리가 되므로, 개발자 본인의 학습·서사 목표(네이티브 경험)를 반영하는 것이 합리적이다 — 이는 순수 기술 판단 외에 프로젝트 목적이 개입하는 지점으로, ADR에 정직하게 남긴다.

### Consequences
- (+) 카메라·로컬 저장·푸시가 안정적. 앱 아이콘 기반 습관화.
- (+) 크로스플랫폼(Compose Multiplatform 등) 채택 시 백엔드(Kotlin)와 언어 통일 — 컨텍스트 스위칭 비용 감소, 포트폴리오상 "하나의 언어로 완결"이라는 서사 확보.
- (−) 스토어 배포·심사 프로세스가 배포 사이클에 추가됨. 1인 개발에서 iOS/Android 동시 대응 시 초기 범위를 Android 우선 등으로 좁힐 필요 검토.
- (−) 보드라이프 링크 유입 시 설치 장벽 발생 → 대응: 랜딩 페이지(정적 웹, 서비스 소개 + 스토어 링크)를 별도로 두어 최초 유입 마찰을 부분 완화. 랜딩은 클라이언트 앱과 무관한 정적 페이지이므로 아키텍처에 영향 없음.

### 뒤집는 트리거
스토어 배포 사이클이 반복 배포 빈도(버그 수정 등)를 심각하게 막는 것으로 판명되면, 웹 클라이언트를 보조 채널로 재검토.
