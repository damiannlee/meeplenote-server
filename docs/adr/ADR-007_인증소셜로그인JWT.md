# ADR-007: 인증 — 소셜 로그인 1종 + 자체 JWT (세션·다중 프로바이더 기각)

**Status:** Accepted / **Date:** 2026-07-09

### Context
US-4.2: 가입 30초 이내. 자격증명 관리(비밀번호)는 1인 운영자가 지기엔 무거운 책임.

### Decision
카카오 로그인 1종(타겟 유저 커버리지 최대)으로 시작. 서버는 provider 토큰 검증 후 자체 JWT(access 30분) + refresh 토큰(DB 저장, 회전) 발급.

### 기각 이유
- 이메일/비밀번호: 재설정 플로우, 유출 책임 — MVP 가치 0에 비용 큼.
- 다중 프로바이더 동시 출시: 계정 연동/병합 문제를 초기부터 안음. 2번째 프로바이더는 "카카오 없는 유저" 항의가 실측될 때.
- 서버 세션: 네이티브 앱은 클라이언트가 서버와 다른 배포 주기로 동작하므로 토큰 방식이 이식성 유리(여러 디바이스, 멀티 플랫폼 대응). 단일 서버라 세션도 가능했으나, refresh 토큰을 DB에 두는 절충으로 강제 로그아웃(탈취 대응) 능력은 유지.

> 구현 시점(M7, 2026-07-11) 참고: refresh 토큰은 재사용형으로 구현됨(매 refresh 호출마다 새로 회전하지 않음) — API 응답이 `{ accessToken }`만 반환하는 확정 스펙(4단계_시스템설계.md §5)에 맞춘 구현 결정. DB 저장은 개별/전체 강제 무효화(탈취 대응) 용도로 `revoked_at` 컬럼을 통해 유지된다.
